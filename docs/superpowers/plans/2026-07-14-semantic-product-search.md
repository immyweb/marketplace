# Semantic Product Search Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Let users find products by typing a natural-language query (e.g. "warm jacket for hiking") instead of only browsing by category/sort, ranked by semantic similarity rather than keyword matching.

**Architecture:** Product text (name + description + category) is embedded with OpenAI's `text-embedding-3-small` at seed time and stored in a `pgvector` column. A search request embeds the query the same way and ranks products by cosine distance via a raw SQL query. The existing `GET /products` endpoint branches to this search path when a `q` param is present; the frontend replaces the category/sort UI with a search box while a query is active.

**Tech Stack:** PostgreSQL + `pgvector` extension, Prisma 7 (`Unsupported` column type + `$queryRaw`/`$executeRaw`), OpenAI Node SDK (`openai` package), Express, Next.js 16 App Router, Vitest + React Testing Library + MSW.

See [the design spec](../specs/2026-07-14-semantic-product-search-design.md) for the full rationale.

## Global Constraints

- Use TypeScript throughout, matching this codebase's existing style (double quotes, no unnecessary comments).
- Every feature needs unit tests. Mocking is fine here — semantic search is not in CLAUDE.md's critical-feature list (checkout/cart/payment/auth), so the embeddings API is mocked in all automated tests. No E2E coverage is needed for this feature (ADR 001).
- **Never auto-commit (CLAUDE.md Rule 6):** every "Stage" step below means `git add` only — do **not** run `git commit`. Leave all changes staged for user review.
- `packages/api` commands run with `packages/api` as the working directory; `packages/web` commands run with `packages/web` as the working directory. `packages/core` must be rebuilt (`bun run --filter @marketplace/core build`) after any change to it before `packages/api` or `packages/web` will see the update — this already happens automatically via each package's `pretest`/`predev` script.
- The embedding model is OpenAI's `text-embedding-3-small`, which returns 1536-dimensional vectors — this dimension is hard-coded into the `vector(1536)` column type and must stay in sync if the model ever changes.
- Semantic search **replaces** the category filter and sort dropdown while a query is active — `q` and `category`/`sort` are never combined, at both the URL-building layer and the API layer.

Add to `packages/api/src/features/products/products.service.ts` (below the existing `listProducts`):

```ts
import { embedText } from "@/shared/embeddings/embeddings.service";

type SearchRow = {
  id: number;
  name: string;
  primary_image: string;
  unit_price: string;
  currency: string;
  category: string;
};

export async function searchProducts(
  q: string,
  page: number,
): Promise<{
  results: ProductDTO[];
  total: number;
  page: number;
  totalPages: number;
}> {
  const embedding = await embedText(q);
  const vector = `[${embedding.join(",")}]`;

  const [rows, countRows] = await Promise.all([
    prisma.$queryRaw<SearchRow[]>`
      SELECT id, name, primary_image, unit_price, currency, category
      FROM products
      WHERE embedding IS NOT NULL
      ORDER BY embedding <=> ${vector}::vector
      LIMIT ${PAGE_SIZE}
      OFFSET ${(page - 1) * PAGE_SIZE}
    `,
    prisma.$queryRaw<
      Array<{ count: bigint }>
    >`SELECT COUNT(*) as count FROM products WHERE embedding IS NOT NULL`,
  ]);

  const total = Number(countRows[0].count);

  return {
    results: rows.map((row) => ({
      ...row,
      unit_price: Number(row.unit_price),
    })),
    total,
    page,
    totalPages: Math.ceil(total / PAGE_SIZE),
  };
}
```

interface ProductsHrefParams {
  page?: number;
  sort?: string;
  category?: string;
  q?: string;
}

export function buildProductsHref({
  page,
  sort,
  category,
  q,
}: ProductsHrefParams) {
  const params = new URLSearchParams();
  if (q) {
    params.set("q", q);
  } else {
    if (page && page > 1) params.set("page", String(page));
    if (sort) params.set("sort", sort);
    if (category) params.set("category", category);
  }
  const qs = params.toString();
  return qs ? `/?${qs}` : "/";
}
```

    if (params?.sort) qs.set("sort", params.sort);

    page: number;

"use client";

import { useEffect, useState } from "react";
import { useRouter, useSearchParams } from "next/navigation";
import { buildProductsHref } from "./product-href";

const DEBOUNCE_MS = 400;

export function ProductSearch() {
  const router = useRouter();
  const searchParams = useSearchParams();
  const [value, setValue] = useState(searchParams.get("q") ?? "");

  useEffect(() => {
    const currentQ = searchParams.get("q") ?? "";
    if (value === currentQ) return;

    const timeout = setTimeout(() => {
      router.push(buildProductsHref({ q: value || undefined }));
    }, DEBOUNCE_MS);

    return () => clearTimeout(timeout);
  }, [value, searchParams, router]);

  return (
    <input
      type="search"
      aria-label="Search products"
      placeholder="Search products…"
      value={value}
      onChange={(e) => setValue(e.target.value)}
      className="mt-6 w-full max-w-sm rounded-full border border-border bg-transparent px-4 py-2 text-sm placeholder:text-muted-foreground focus:border-primary focus:outline-none"
    />
  );
}

import type { Metadata } from "next";
import { fetchProducts } from "@/lib/api";
import {
  ProductCard,
  ProductFilters,
  ProductSearch,
  Pagination,
} from "@/app/products/_components";

export const metadata: Metadata = {
  title: "Shop All Products",
  description: "Browse our full range of clothing and accessories.",
  openGraph: {
    title: "Shop All Products",
    description: "Browse our full range of clothing and accessories.",
  },
};

interface Props {
  searchParams: Promise<{
    page?: string;
    sort?: string;
    category?: string;
    q?: string;
  }>;
}

export default async function ProductListingPage({ searchParams }: Props) {
  const { page, sort, category, q } = await searchParams;
  const currentPage = page ? Number(page) : 1;
  const { results, total, totalPages } = await fetchProducts({
    page: currentPage,
    sort,
    category,
    q,
  });

  return (
    <>
      <p className="font-mono text-xs tracking-widest text-muted-foreground uppercase">
        Full Catalog · {total} Goods
      </p>
      <h1 className="mt-1 text-2xl">All Products</h1>
      <ProductSearch />
      {q ? null : <ProductFilters activeCategory={category} sort={sort} />}
      {results.length === 0 ? (
        <p className="mt-8 text-muted-foreground">
          {q
            ? `No results for "${q}".`
            : category
              ? "No products in this category."
              : "No products available."}
        </p>
      ) : (
        <>
          <ul
            aria-label="Product listing"
            className="mt-8 grid list-none grid-cols-2 gap-x-6 gap-y-10 p-0 sm:grid-cols-3 lg:grid-cols-4"
          >
            {results.map((product, index) => (
              <li key={product.id}>
                <ProductCard {...product} eager={index < 3} />
              </li>
            ))}
          </ul>
          <Pagination
            page={currentPage}
            totalPages={totalPages}
            sort={sort}
            category={category}
          />
        </>
      )}
    </>
  );
}
```

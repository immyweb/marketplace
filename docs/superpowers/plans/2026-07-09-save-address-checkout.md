# Save Address at Checkout Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Let a signed-in user save their checkout delivery address so it prefills (still editable) on future checkout visits.

**Architecture:** Four nullable address columns added directly to the existing `User` Prisma model (no new table — single address per user, product decision). A new `GET /account/address` endpoint reads it; `POST /order` gains a `saveAddress: boolean` flag that, when true, upserts those columns inside the same transaction as order creation. The Next.js checkout page fetches the saved address server-side and passes it into the existing checkout form as `defaultValues`; a new checkbox (default checked) controls the flag sent with the order.

**Tech Stack:** TypeScript, Express + Prisma 7 (`packages/api`), Next.js App Router + react-hook-form + zod (`packages/web`), Vitest + Supertest (API tests, real Postgres), Vitest + RTL + MSW (web component tests), Playwright (E2E).

## Global Constraints

- Full spec: `docs/superpowers/specs/2026-07-09-save-address-checkout-design.md`. Read it before starting if anything below is ambiguous.
- TypeScript throughout (CLAUDE.md rule 7).
- API tests for checkout/order code run against real Postgres and real Stripe test-mode calls — no mocks for these (CLAUDE.md rule 7, existing `orders.test.ts` convention). Test DB is `marketplace_test`, wired via `packages/api/tests/setup.ts`; Docker must be running (`docker compose up -d` from the repo root) before any API test or migration command.
- Web: favor component tests (Vitest + RTL + MSW) per ADR 001; checkout is a designated critical flow, so this feature also gets one E2E scenario extending the existing `checkout.spec.ts` — no new E2E spec file.
- **Never auto-commit** (CLAUDE.md rule 6, overrides this skill's default per-task commit step): each task below ends once its tests pass. Do not run `git commit` — leave the working tree uncommitted for user review.
- New `User` columns are camelCase (`addressName`, not `address_name`), matching every other column already on that Better Auth–managed table (`emailVerified`, `createdAt`) — not `Order`'s snake_case convention, since these columns live on `User`, not a hand-authored domain table.
- `saveAddress` is a required field on `PlaceOrderSchema` (no server-side default) — the client always sends it explicitly, tied to the checkout checkbox state.

```ts
import type { AddressDetails } from "@marketplace/core";
import { prisma } from "@/shared/db/prisma";

export async function getSavedAddress(
  userId: string,
): Promise<AddressDetails | null> {
  const user = await prisma.user.findUniqueOrThrow({
    where: { id: userId },
    select: {
      addressName: true,
      addressStreet: true,
      addressCity: true,
      addressPostcode: true,
    },
  });

  if (!user.addressName) return null;

  return {
    name: user.addressName,
    street: user.addressStreet!,
    city: user.addressCity!,
    postcode: user.addressPostcode!,
  };
}
```

```tsx
  const [saveAddress, setSaveAddress] = useState(true);

  const {
    register,
    handleSubmit,
    formState: { errors },
  } = useForm<CheckoutFormValues>({
    resolver: zodResolver(AddressSchema),
    defaultValues: savedAddress ?? undefined,
  });

  async function onSubmit(values: CheckoutFormValues) {
    if (!stripe || !elements || !cart.id) return;
    setSubmitting(true);
    setFormError(null);

    try {
      const { clientSecret } = await createPaymentIntent(cart.id);

      const cardElement = elements.getElement(CardElement);
      if (!cardElement) throw new Error("Card element not mounted");

      const { error, paymentIntent } = await stripe.confirmCardPayment(
        clientSecret,
        {
          payment_method: {
            card: cardElement,
            billing_details: { name: values.name, address: { country: "GB" } },
          },
        },
      );

      if (error || !paymentIntent) {
        setFormError(error?.message ?? "Payment failed. Please try again.");
        return;
      }

      const order = await placeOrder({
        cartId: cart.id,
        paymentIntentId: paymentIntent.id,
        address_details: values,
        saveAddress,
      });

      router.push(`/order-confirmation/${order.id}`);
    } catch (err) {
      setFormError(err instanceof Error ? err.message : "Something went wrong");
    } finally {
      setSubmitting(false);
    }
  }

  return (
    <form
      onSubmit={handleSubmit(onSubmit)}
      aria-label="Checkout form"
      noValidate
      className="max-w-2xl"
    >
      <h1 className="text-2xl">Checkout</h1>

      <AddressForm register={register} errors={errors} />

      <label className="mt-4 flex items-center gap-2 text-sm">
        <input
          type="checkbox"
          checked={saveAddress}
          onChange={(e) => setSaveAddress(e.target.checked)}
        />
        Save this address for future orders
      </label>

      <StripePaymentForm />

      <div className="mt-8 flex items-center justify-between border-t-2 border-primary pt-4">
        <p
          aria-label={`Order total: £${cart.total_price.toFixed(2)}`}
          className="font-display text-lg font-bold tracking-wide uppercase"
        >
          Total: £{cart.total_price.toFixed(2)}
        </p>
        <Button
          type="submit"
          disabled={submitting || !stripe}
          aria-busy={submitting}
        >
          {submitting ? "Processing..." : "Place Order"}
        </Button>
      </div>

      {formError && (
        <p role="alert" className="mt-4 text-sm text-destructive">
          {formError}
        </p>
      )}
    </form>
  );
}

export function CheckoutFormPage({
  savedAddress = null,
}: {
  savedAddress?: AddressDetails | null;
}) {
  const [cart, setCart] = useState<Cart | null>(null);
  const router = useRouter();

  useEffect(() => {
    fetchCart().then((c) => {
      if (!c.id || c.items.length === 0) {
        router.push("/cart");
        return;
      }
      setCart(c);
    });
  }, [router]);

  if (!cart) return <p aria-busy="true">Loading...</p>;

  return (
    <Elements stripe={stripePromise}>
      <CheckoutForm cart={cart} savedAddress={savedAddress} />
    </Elements>
  );
}
```

# Footer Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a site-wide footer with links to six policy pages (terms, privacy, cookies, sustainability, accessibility, reviews policy), and stub out each of those six pages.

**Architecture:** A new `Footer` Server Component, styled to match the nav, rendered in `app/layout.tsx` after `{children}`. Six new static route pages, each following the existing plain-page convention (`app/orders/page.tsx`'s `<h1>` + body shape), with placeholder copy only.

**Tech Stack:** Next.js (Server Components, App Router), React, Tailwind CSS, Vitest + React Testing Library for component tests.

## Global Constraints

- Design spec: `docs/superpowers/specs/2026-07-09-footer-design.md`.
- Footer visually matches the nav: `bg-primary text-primary-foreground`, same `mx-auto max-w-6xl px-4 py-4 sm:px-6` rhythm as `components/nav.tsx`'s `<nav>` element, links styled `font-mono text-sm tracking-wide uppercase hover:text-accent`.
- No policy content — stub pages only, each with a heading and a single "Content coming soon." placeholder line.
- No E2E test — not a checkout/cart/payment/auth-critical flow (ADR 001).
- This repo's CLAUDE.md overrides the "commit" step below: leave changes uncommitted for user review, do not `git add`/`git commit`.

- [ ] **Step 3: Create the `Footer` component**

Create `packages/web/components/footer.tsx`:

```tsx
import Link from "next/link";

const FOOTER_LINKS = [
  { href: "/terms", label: "Terms & Conditions" },
  { href: "/privacy", label: "Privacy Notice" },
  { href: "/cookies", label: "Cookies" },
  { href: "/sustainability", label: "Sustainability" },
  { href: "/accessibility", label: "Accessibility" },
  { href: "/reviews-policy", label: "Reviews Policy" },
] as const;

export function Footer() {
  return (
    <footer className="bg-primary text-primary-foreground">
      <div className="mx-auto flex max-w-6xl flex-wrap gap-x-6 gap-y-2 px-4 py-4 sm:px-6">
        {FOOTER_LINKS.map((link) => (
          <Link
            key={link.href}
            href={link.href}
            className="font-mono text-sm tracking-wide uppercase hover:text-accent"
          >
            {link.label}
          </Link>
        ))}
      </div>
    </footer>
  );
}
```

- [ ] **Step 4: Run the test to verify it passes**

Run: `cd packages/web && npx vitest run tests/component/footer.test.tsx`
Expected: PASS.

- [ ] **Step 5: Wire `Footer` into the root layout**

In `packages/web/app/layout.tsx`, add the import and render `<Footer />` after `{children}`. Full new contents:

```tsx
import "./globals.css";
import type { Metadata } from "next";
import { Big_Shoulders, IBM_Plex_Mono, Public_Sans } from "next/font/google";
import { Nav } from "@/components/nav";
import { Footer } from "@/components/footer";

const display = Big_Shoulders({
  subsets: ["latin"],
  weight: ["600", "700", "800"],
  variable: "--font-heading",
});

const body = Public_Sans({
  subsets: ["latin"],
  weight: ["400", "500", "600", "700"],
  variable: "--font-body",
});

const mono = IBM_Plex_Mono({
  subsets: ["latin"],
  weight: ["400", "500"],
  variable: "--font-code",
});

export const metadata: Metadata = {
  title: { default: "Marketplace", template: "%s | Marketplace" },
  description: "Quality clothing and accessories, delivered to your door.",
  openGraph: {
    siteName: "Marketplace",
    locale: "en_GB",
    type: "website",
  },
};

export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html
      lang="en"
      className={`${display.variable} ${body.variable} ${mono.variable}`}
    >
      <body>
        <Nav />
        <main
          id="main-content"
          className="mx-auto max-w-6xl px-4 py-10 sm:px-6"
        >
          {children}
        </main>
        <Footer />
      </body>
    </html>
  );
}
```
